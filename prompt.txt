# FONCTIONNALITÉS PRINCIPALES

## 1. Upload d’image

Créer une zone drag & drop premium permettant d’importer une image.

Formats acceptés :
- JPG
- JPEG
- PNG
- WEBP
- éventuellement AVIF si support navigateur disponible

L’image importée doit être affichée immédiatement dans l’interface.

Prévoir :
- aperçu de l’image,
- taille du fichier,
- dimensions,
- type MIME,
- nom du fichier,
- date éventuelle si disponible.

---

## 2. Analyse des métadonnées

L’application doit tenter d’extraire et afficher les informations disponibles :

- nom du fichier,
- poids,
- format,
- largeur,
- hauteur,
- ratio,
- date de dernière modification,
- type MIME,
- présence ou absence de métadonnées visibles,
- indication si les métadonnées semblent absentes ou supprimées.

Si possible en JavaScript pur, lire les métadonnées EXIF de base.
Sinon prévoir une structure extensible et documenter les limites.

Afficher une section :

“Metadata & Provenance”

Avec des indicateurs :
- EXIF trouvé : oui/non
- appareil photo détecté : oui/non
- logiciel détecté : oui/non
- C2PA / Content Credentials : détecté / non détecté / non supporté dans cette version
- suspicion de suppression de métadonnées : faible / moyenne / forte

Important :
Ne jamais conclure automatiquement qu’une absence de métadonnées signifie IA.
Afficher :
“L’absence de métadonnées n’est pas une preuve. Beaucoup de plateformes suppriment automatiquement ces données.”

---

## 3. Analyse visuelle probabiliste

Créer un moteur d’analyse simulé mais structuré, prêt à être connecté plus tard à une vraie API ou à un vrai modèle.

Le système doit analyser plusieurs familles d’indices :

### Indices visage / corps
- peau trop lisse,
- yeux incohérents,
- dents ou bouche anormales,
- mains ou doigts suspects,
- oreilles asymétriques,
- lunettes/reflets incohérents.

### Indices scène
- arrière-plan flou ou incohérent,
- perspective anormale,
- ombres incohérentes,
- lumière non homogène,
- objets fusionnés,
- texte illisible ou déformé.

### Indices techniques
- bruit numérique incohérent,
- compression étrange,
- zones trop propres,
- artefacts de génération,
- répétitions de motifs.

Créer un score par catégorie :
- faible,
- moyen,
- élevé.

Puis un score global :
AI Probability Score : 0 à 100

Exemple :
- 0 à 30 : probablement réel
- 31 à 60 : incertain / retouché possible
- 61 à 80 : probablement généré ou fortement modifié par IA
- 81 à 100 : forte probabilité IA

Attention :
Le score doit être présenté comme une estimation heuristique.
Ne pas présenter une vérité absolue.

---

## 4. Analyse forensique visuelle

Créer une section “Forensic Map”.

Afficher une zone d’analyse avec :
- aperçu image,
- grille légère,
- zones suspectes simulées sous forme de heatmap,
- surlignage des zones potentiellement incohérentes,
- légende.

Même si la première version est heuristique ou simulée, l’architecture doit être prête pour brancher plus tard :
- OpenCV,
- modèle PyTorch,
- API de détection,
- analyse ELA,
- analyse bruit/compression.

Créer un commentaire clair :
“Cette carte met en évidence les zones nécessitant une vérification humaine. Elle ne constitue pas une preuve définitive.”

---

## 5. Résultat final

Créer une carte centrale de résultat avec :

- Verdict :
  - Probably Authentic
  - Probably AI-Generated
  - Possibly Edited
  - Inconclusive

- AI Probability Score
- Confidence Level :
  - Low
  - Medium
  - High

- Risk Level :
  - Low
  - Moderate
  - High
  - Critical

- Résumé automatique :
  Exemple :
  “L’image présente plusieurs indices compatibles avec une génération IA : métadonnées absentes, texture de peau anormalement uniforme, incohérences de lumière et artefacts dans l’arrière-plan. Le résultat reste probabiliste.”

“Important Limitations”

Texte recommandé :

Une image compressée, recadrée, retouchée ou capturée via screenshot peut réduire fortement la fiabilité de l’analyse. L’absence de métadonnées ne constitue pas une preuve de génération IA. Seule une signature de provenance fiable, comme certains mécanismes C2PA ou Content Credentials, peut apporter une preuve plus forte d’origine ou de modification.”
